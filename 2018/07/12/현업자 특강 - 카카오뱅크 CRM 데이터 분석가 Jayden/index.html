CRM에서 뭐 하세요?

Customer relationship management (CRM)

고객과 회사간의 Interaction을 찾아서 하는 것
ex) Targeting 광고 문자 받기, 특정 웹 사이트 들어가면 Target화 된 광고 문구 보는 것

CRM 에서는 Data 분석이 전통적으로 필요하다. 특히 금융권에서는 데이터 분석이라고 하면 CRM과 CSS쪽이 유명이다.

1. CRM Mission?

고객의 가치를 아는 것(고객 가치 평가 모델 개발) + 고객의 가치를 증대 시키는 것(CRM 프로그램 기획/운영)

2. CRM에서 고객을 보는 관점

신규 고객 관리 및 기존 고객 관리

3. CRM 워크플로우

데이터 기획 - 데이터 수집 - 데이터 분석 - 분석 리포트
데이터 수집 - 분석 마트 설계 - 분석 마트 구축 - 고객 행동 모형 개발 - 개인화 서비스 기획 - 개인화 서비스 구현
고객 행동 모형 개발 - 개인화 마케팅 기획 - 개인화 마케팅 실행

일반 글자 : CRM

자주 측정하는 형태의 데이터는 Mart라고 해서 고객이 자주 쓰는 항목을 Table화한다.

4. CRM 데이터 분석 단계

  1. 업무영역별 Driving Table 선정
  1. 분석 필요 Input 정보 추가
  1. Pivot, Summary 정보 추가
  1. 업무영역별 분석 Table 통합
  1. Cutoff_Data 기준 Data set 작업
  1. EDA 마트 구성
  1. 못 적었다..! 아무튼 마무리 단계

5. CRM 데이터 분석 단계

  1. 분석 목적 Comm. : 정기 예금 이탈 예측
  1. Target Feature 정의 : 가장 오랫동안 들이는 시간, 여기서는 만기 예금 재예치 또는 만기 이후 30일 이내에 새로운 예금 가입을 하지 않은 고객
  1. Target EDA : 정기 예금 가입/이탈 고객 특성 탐색 및 분석 모수 선정
  1. Data Pre-process : 연속성 변수 구간화, 원핫인코딩 등 분석에 필요한 각종 데이터 수집 및 분석에 용이한 형태의 데이터셋 생성
  1. Feature Engineering : 700개 가량의 파생 변수가 생성됨(실제 모델에서는 훨씬 적게 사용함)
  1. Modeling : Decision Tree, Logistic Regression, Random Forest, SVM 등
  1. Model Selection : Logistic Regression
  1. Score 적재 : CRM에서 가장 중요하게 여기는 부분은 비용에 대한 Control. 저비용, 고효율을 Moto로 하기 때문임.
  1. 활용

Feature Engineering is Everything!
